# ADR-0006: Діалоговий нативний оркестратор `/spec-forge` (без API-ключа)

**Status:** accepted
**Date:** 2026-07-09
**Deciders:** Developer

## Context
Тонка обгортка `/spec-forge` просто запускала CLI, а зміст ішов через Anthropic Messages API
(ADR-0002) → потрібні платні API-кредити, яких у користувача немає (лише **Claude Max**-підписка).
Потрібен нативний діалоговий флоу, що будує спеку прямо в Claude Code.

## Decision
- Переписати `_WRAPPER` на **діалоговий оркестратор**: інтервʼю у головному треді → делегування
  чернеток рольовим субагентам через `Task` → файли у нашій структурі `specifications/`.
  Фази **BA → SA** + опційні **Designer / Developer**, з людським гейтом між кожною.
- **Самодостатній runtime:** під час флоу — без викликів CLI і без `ANTHROPIC_API_KEY`
  (усе на Claude Max-підписці).
- Встановлювати **5 рольових субагентів** у `~/.claude/agents/` (щоб делегування працювало в
  будь-якому проєкті); джерело — `spec_forge/templates/bundle/ai/agents/*.md`.
- **Self-upgrade** команди за версією-маркером (`<!-- spec-forge-command vN -->`);
  `command install --force` оновлює команду + агентів.
- Розширює ADR-0004 (авто-provision). Messages-API шлях (ADR-0002) лишається для CI/автоматизації
  (`--backend claude`).

## Consequences
**Позитивні**
- Працює **без API-кредитів** (на підписці); зміст генерується нативно рольовими персонами.
- Субагенти доступні глобально — флоу працює в будь-якому проєкті.

**Негативні / компроміси**
- У цьому флоу немає детермінованого scaffold / validate / export (за рішенням користувача) —
  вони лишаються в CLI окремо (`init`/`validate`/`export`).
- Установка команди/агентів усе ще через `command install` / auto-callback (одноразово); окремий
  Claude Code plugin (plugin.json + marketplace) — поза скоупом.
- Субагенти неінтерактивні (окремий тред) → інтервʼю в головному треді, делегування з повним брифом.
